Documentación

Bienvenido a la sección de documentación. Aquí encontrarás guías y tutoriales para utilizar RAPITEST y realizar solicitudes API eficientemente.

RAPITEST es una herramienta fácil de usar para probar APIs. Sigue estos pasos:

  1. Escribe el endpoint (URL) de tu API en el campo de texto proporcionado.
  2. Selecciona el método HTTP (GET, POST, PUT, DELETE) desde el menú desplegable.
  3. Si es necesario, escribe el cuerpo de la solicitud en formato JSON en el cuadro correspondiente.
  4. Presiona el botón "Enviar Solicitud" para obtener la respuesta de la API.
  5. Consulta el historial para revisar solicitudes anteriores.

El método GET se utiliza para obtener datos de un servidor. Ejemplo:

URL: https://jsonplaceholder.typicode.com/posts
Método: GET
Respuesta esperada: Lista de publicaciones en formato JSON.

El método POST se utiliza para enviar datos a un servidor y crear un nuevo recurso. Ejemplo:

URL: https://jsonplaceholder.typicode.com/posts
Método: POST
Cuerpo:
{
    "title": "Nuevo post",
    "body": "Este es el contenido del post.",
    "userId": 1
}
Respuesta esperada: Un objeto JSON con el nuevo recurso creado.

El método PUT se utiliza para actualizar recursos existentes en el servidor. Ejemplo:

URL: https://jsonplaceholder.typicode.com/posts/1
Método: PUT
Cuerpo:
{
    "id": 1,
    "title": "Título actualizado",
    "body": "Contenido actualizado.",
    "userId": 1
}
Respuesta esperada: Un objeto JSON con los datos actualizados.

El método DELETE se utiliza para eliminar un recurso del servidor. Ejemplo:

URL: https://jsonplaceholder.typicode.com/posts/1
Método: DELETE
Respuesta esperada: Un objeto vacío {} indicando que el recurso fue eliminado.
